First

The original XBOX went through several revisions that range from 1.0 - 1.6b. How to identify the mainboard revision is a good place to start. There are a few aesthetic clues that can help identify the revision before even cracking it open. For example, the year made on the barcode sticker, and the shape of the DVD tray.

Next, check the clock capacitor. The clock capacitors on the original XBOX are known to deteriorate and leak acid which can cause damage to the PCB and surrounding components. Microsoft apparently broke the norm by using a capacitor instead of a lithium coin cell battery to hold clock time. On most mainboard revisisons it is advisable to remove the capacitor completely then clean up any leaked acid. Doing so does not affect functionality.

Softmod

The goal of a softmod is to gain privilages through software to kernel mode memory which then would make it possible to run software that was not originally intended by the manufacturer. Once that is accomplished, persistance is obtained by installing a modified shell. Basically the shell in XBOX terms is called a dashboard. There are quite a few guides out there on how to softmod (some of which require more complex steps than others). A bit of obscure equipment is necessary including a 512 MB USB thumb drive, and a USB XBOX controller adapter. Apparently an OG XBOX controller is basically a USB peripheral!

The next mod to consider is a HDD upgrade. A few additional pieces of special equipment are required. The softmod method requires a little practice and luck because it involves hot-swapping the optical drive for the new HDD before the XBOX notices.

  • https://www.youtube.com/watch?v=lBC9599ww6I

A softmod is all that is necessary to add tons of useability to an original XBOX. I categorize most of the modding I have talked about so far as a softmod because no soldering has been necessary yet.

Hardmod

This is where some skill is involved and things get interesting.

The first step to upgrading RAM is loading a firmware/BIOS/kernel that is compatible with variable RAM quantity (the stock BIOS is not). In XBOX terms firmware, BIOS, kernel, TSOP, EEPROM are all the same thing because the kernel is included on the same chip as the BIOS. Loading a 3rd party BIOS can be achieved in a few ways. The first would be to reconnect the write protection circut of the TSOP and flash a new firmware directly onto the built-in ROM. The second method is to install a modchip which typically stores multiple BIOS images that can be loaded as needed.

Disabling write protection on the TSOP is different given the mainboard revision. On the 1.0 in this case there are two solder points to join. R7D3 on the top and R7R4 on the bottom. When that is done it is as simple as writing XBlast_OS_v0.56 onto the TSOP. The XBlast_OS BIOS allows the XBOX to boot with variable RAM quantities and a test feature for each RAM chip as you go.
