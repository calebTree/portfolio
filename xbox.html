First

There are a few beginner things to be aware of when one first starts to mod an original XBOX. How to identify the mainboard revision is a good place to start. Because the secret ROM was cracked so quickly OG XBOX mainboards went through several revisions that range from 1.0 - 1.6b. To start, interestingly, there are a few external clues that can be used to help identify the mainboard version without the need to open it up. For example, by looking at the year made on the barcode sticker, and the shape of the DVD tray. The next thing to look at is to check the clock capacitor, the clock capacitors on these are known to deteriorate. Also XBOX broke the norm by using a capacitor instead of a lithium coin cell battery to hold clock time. On most mainboard revisisons it is advisable to remove the capacitor completely and this does not affect functionality.

The Softmod

Eventually I stumbled on a simple guide illustrating how to perform the softmod. The goal of a softmod is essentially to gain privilages through software to kernel mode operating system code that will make it possible to run software that was not originally intended by the manufacturer. The ultimate goal of an XBOX softmod is to install a persistant modified operating system. The shell to the operating system in XBOX terms is called the dashboard. There are quite a few guides out there on how to softmod (some of which require more complex steps than others). A bit of obscure equipment necessary was a 512 MB USB thumb drive which I luckily already had, and a USB XBOX controller adapter. Apparently an OG XBOX controller is essentially a modified USB device. Although there are other possible ways to softmod an XBOX; I have found this "XBOX Softmodding Tool" to be the most mainstreamed. I do have one warning before softmodding. The afformentioned tool comes with a function to reverse the softmod - nearly bringing your XBOX back to a factory state. However, it does leave behind a file that I have yet to find a way to delete from the restored manufacturer's dashboard. If you have concluded that it is worth it to sftmod there are a couple of steps to bring the hacked savegame files over to the XBOX via your new equipment I mentoned above but once that is complete you'll find your XBOX softmodded before you know it.

The first thing to do with the softmodded XBOX is familiarize yourself with the partition and file system structure. There is at least a C, an E and X, Y, Z partition. More on F and G partitions later. C and E are user partitions but X, Y, Z are used by the system for cache. There are also UDATA and TDATA folders. Those folders store settings and save files for games. It took me a while to get used to how the partitions are setup. In each partition there can be folders named "Applications, Games, Emulation, Dashboard" etc. It doesn't matter to the shell (I like UnleashX) which partition the folder is on; all of the similarly named folders will appear to the shell as one. The filesystem and system files look a little chaotic at first. Most dashboards have a "File Explorer" type app to brows folders and files. It provides all the necessary functions one would expect from a file system through the human interface of the controller. To accomodate a controller interface the File Explorer is split down the middle. Both sides are the same file system. To move or copy a file or directory navigate to the file of interest on one side (either side) then to the destination on the opposite side, press start and make your selection. The opposite side from the selection is interpreted as the destination.

The Hardmod

This is where some skill is involved and things get interesting. I couldn't find a good reason to hardmod until I found the original Half-Life homebrew on XBMC-Emustation and learned that it comes in two versions. A 64MB version (stock XBOX) where complex textures have been deleted to accomodate limited memory. As well as the 128 MB version which works on units that have been hardmodded with double the RAM.

The first thing that has to be done to upgrade the RAM on an XBOX is load a firmware/BIOS/kernel (in XBOX modding community lingo these are all the same thing) that is compatible with various RAM capacity (the stock BIOS is not). That can be done one of two ways. The first slightly more risky method would be to short the write protection out to the built in ROM called the TSOP and flash a new firmware onto that. The second lesss risky method is to install a modchip which stores multiple BIOS images that can be loaded on demand and this way the built in BIOS does not have to be tampered with.

To disable write protectionn on the 1.0 mainboard revision there are two solder points to join. R7D3 on the top and R7R4 on the bottom. When that is done it is as simple as writing XBlast_OS_v0.56 onto the TSOP. The XBlast_OS BIOS allows the XBOX to boot with varying RAM quantities which would normally cause it to fail to boot. When the BIOS is flashed simply soldered the RAM onto each open position testing each one using XBlast_OS as you go.
